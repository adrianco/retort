# Architecture Summary

MCP server for Brazilian soccer data (TypeScript, ESM, vitest).

- `index.ts` — entrypoint: builds `DataLoader` over `data/kaggle/`, connects `McpServer` over stdio.
- `loader.ts` — reads all 6 Kaggle CSVs (csv-parse), normalizes them into a deduped `NormalizedMatch[]` with source priority (primary sources > historical pre-2012 > extended for uncovered years); also loads FIFA players.
- `normalizer.ts` — team-name normalization (state-suffix stripping, diacritics-insensitive fuzzy match), multi-format date parsing (ISO + DD/MM/YYYY).
- `queries.ts` — match search (team/either, competition, season, date range), team stats (W/L/D, goals, points), head-to-head, standings (computed from results), aggregate statistics.
- `players.ts` — player search by name/nationality/club/position/rating.
- `tools.ts` — formats query results into text; dispatches the 6 MCP tools.
- `server.ts` — registers 6 MCP tools with zod schemas: search_matches, get_team_stats, search_players, get_standings, get_statistics, head_to_head.
